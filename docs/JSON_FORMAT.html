Gen Runtime JSON Patch Format Specification ​

Version: 1.0 Status: Draft Last Updated: 2025-10-31

Overview ​

This document defines the JSON format for Gen synthesizer patches. This format is used by:

  • The web-based patch editor to save/load patches
  • The ESP32 runtime interpreter to configure the synthesizer
  • Patch sharing and version control

Top-Level Structure ​

json
{
  "version": "1.0",
  "name": "Patch Name",
  "description": "Optional description",
  "author": "Optional author name",
  "created": "2025-10-31T12:00:00Z",
  "clockSpeed": 240,
  "clockMode": 1,
  "modules": [ /* array of module objects */ ],
  "connections": [ /* array of connection objects */ ]
}

Fields ​

  • version (string, required): Format version for future compatibility
  • name (string, required): Human-readable patch name
  • description (string, optional): Patch description/notes
  • author (string, optional): Patch creator
  • created (string, optional): ISO 8601 timestamp
  • clockSpeed (number, optional): CPU frequency in MHz (240, 280, 320, 360, 400). Default: 240
  • clockMode (number, optional): Clock mode - 0=FIXED, 1=AUTO, 2=POWERSAVE. Default: 0 (FIXED)
  • modules (array, required): List of module instances
  • connections (array, required): List of audio/control connections

Performance Settings ​

Clock Speed Options:

  • 240 - Default safe speed, works on all hardware
  • 280 - +17% performance (safe on v1.0 hardware with bypassed polyfuse)
  • 320 - +33% performance (recommended for complex patches)
  • 360 - +50% performance (test stability first)
  • 400 - +67% performance (may be unstable, v1.1+ recommended)

Clock Modes:

  • 0 (FIXED) - Run at specified clockSpeed always
  • 1 (AUTO) - Automatically adjust frequency based on CPU load (recommended)
  • 2 (POWERSAVE) - Prioritize low power, only increase speed when necessary

Module Object Structure ​

json
{
  "id": "osc1",
  "type": "Oscillator",
  "params": {
    /* type-specific parameters */
  },
  "position": {
    "x": 100,
    "y": 200
  }
}

Module Fields ​

  • id (string, required): Unique identifier for this module instance

    • Format: type + number (e.g., "osc1", "filter2")
    • Must be unique within the patch
    • Used for connection routing

  • type (string, required): Module type identifier

    • Must match a supported module type (see Module Types below)
    • Case-sensitive

  • params (object, required): Module-specific parameters

    • Structure varies by module type
    • All parameters must have default values in runtime

  • position (object, optional): UI position for visual editor

    • x (number): X coordinate in canvas
    • y (number): Y coordinate in canvas
    • Not used by ESP32 runtime

Connection Object Structure ​

json
{
  "from": {
    "module": "osc1",
    "output": "outL"
  },
  "to": {
    "module": "filter1",
    "input": "inL"
  }
}

Connection Fields ​

  • from (object, required): Source endpoint

    • module (string): Source module ID
    • output (string): Output jack name

  • to (object, required): Destination endpoint

    • module (string): Destination module ID
    • input (string): Input jack name

Module Types & Parameters ​

Audio Sources ​

Oscillator ​

json
{
  "id": "osc1",
  "type": "Oscillator",
  "params": {
    "waveform": "saw",        // "sine", "saw", "square", "triangle"
    "frequency": 440.0,       // Hz (float)
    "detune": 0.0,           // semitones (float)
    "pulseWidth": 0.5,       // 0.0-1.0 (for square wave)
    "level": 1.0             // 0.0-1.0 output gain
  }
}

Outputs: outL, outR (stereo) Inputs: freqMod (CV), pwmMod (CV)

SamplePlayer ​

json
{
  "id": "sample1",
  "type": "SamplePlayer",
  "params": {
    "samplePath": "/samples/kick.wav",  // Path on SD/SPIFFS
    "playbackRate": 1.0,                // Speed multiplier
    "loop": false,                       // Boolean
    "startPoint": 0.0,                  // 0.0-1.0
    "endPoint": 1.0,                    // 0.0-1.0
    "level": 1.0                        // 0.0-1.0
  }
}

Outputs: outL, outRInputs: trigger (gate), pitchMod (CV)

Audio Processors ​

Filter ​

json
{
  "id": "filter1",
  "type": "Filter",
  "params": {
    "filterType": "lowpass",  // "lowpass", "highpass", "bandpass", "notch"
    "cutoff": 1000.0,         // Hz (float)
    "resonance": 0.7,         // 0.0-1.0
    "gain": 1.0               // 0.0-2.0 (for some filter types)
  }
}

Outputs: outL, outRInputs: inL, inR, cutoffMod (CV), resMod (CV)

VCA ​

json
{
  "id": "vca1",
  "type": "VCA",
  "params": {
    "gain": 1.0,              // 0.0-2.0
    "mode": "linear"          // "linear" or "exponential"
  }
}

Outputs: outL, outRInputs: inL, inR, cv (CV control input)

Delay ​

json
{
  "id": "delay1",
  "type": "Delay",
  "params": {
    "delayTime": 500.0,       // milliseconds
    "feedback": 0.5,          // 0.0-1.0
    "mix": 0.3,               // 0.0-1.0 (dry/wet)
    "stereoOffset": 0.0       // milliseconds (L/R delay difference)
  }
}

Outputs: outL, outRInputs: inL, inR, timeMod (CV), fbMod (CV)

Reverb ​

json
{
  "id": "reverb1",
  "type": "Reverb",
  "params": {
    "roomSize": 0.7,          // 0.0-1.0
    "damping": 0.5,           // 0.0-1.0
    "mix": 0.3                // 0.0-1.0 (dry/wet)
  }
}

Outputs: outL, outRInputs: inL, inR

Control Sources ​

ADSR ​

json
{
  "id": "adsr1",
  "type": "ADSR",
  "params": {
    "attack": 10.0,           // milliseconds
    "decay": 100.0,           // milliseconds
    "sustain": 0.7,           // 0.0-1.0 level
    "release": 200.0          // milliseconds
  }
}

Outputs: out (CV) Inputs: gate (gate signal)

LFO ​

json
{
  "id": "lfo1",
  "type": "LFO",
  "params": {
    "waveform": "sine",       // "sine", "saw", "square", "triangle"
    "rate": 2.0,              // Hz
    "depth": 1.0,             // 0.0-1.0
    "phase": 0.0              // 0.0-1.0 (starting phase)
  }
}

Outputs: out (CV) Inputs: rateMod (CV)

Hardware I/O ​

Pot ​

json
{
  "id": "pot1",
  "type": "Pot",
  "params": {
    "potNumber": 1,           // 1-4 (which physical pot)
    "min": 0.0,               // Output range minimum
    "max": 1.0,               // Output range maximum
    "curve": "linear",        // "linear", "exponential", "logarithmic"
    "smoothing": 0.1          // 0.0-1.0 (low-pass filtering)
  }
}

Outputs: out (CV) Inputs: None

MIDIIn ​

json
{
  "id": "midi1",
  "type": "MIDIIn",
  "params": {
    "channel": 1,             // 1-16 (0 = omni)
    "voiceMode": "mono"       // "mono" or "poly"
  }
}

Outputs: gate, pitch (CV), velocity (CV), modwheel (CV) Inputs: None

I2SOut ​

json
{
  "id": "out1",
  "type": "I2SOut",
  "params": {
    "gain": 1.0               // Master output level 0.0-1.0
  }
}

Outputs: None (physical DAC output) Inputs: inL, inR

Validation Rules ​

Module Validation ​

  1. All module IDs must be unique
  2. All module types must be supported by runtime
  3. All required parameters must be present
  4. Parameter values must be within valid ranges

Connection Validation ​

  1. Source and destination modules must exist
  2. Output/input jack names must be valid for module types
  3. No circular dependencies (topological sort must succeed)
  4. Audio jacks connect to audio jacks, CV to CV

Size Limits (Runtime Constraints) ​

  • Maximum modules: 50 (configurable in firmware)
  • Maximum connections: 200
  • Maximum JSON file size: 32KB (SPIFFS limit)
  • Module ID max length: 16 characters
  • Parameter string max length: 64 characters

Version Compatibility ​

Version 1.0 (Current) ​

  • Initial specification
  • Core module types
  • Basic parameter support

Future Versions ​

  • Version 1.1: Add custom wavetables
  • Version 1.2: Add sequencer patterns
  • Version 2.0: Extended parameter automation

Runtime must handle forward compatibility:

  • Unknown parameters → use defaults
  • Unknown module types → skip with warning
  • Higher version number → load if possible, warn if incompatible

Notes for Developers ​

Web App Considerations ​

  • Validate JSON before upload to avoid ESP32 parsing errors
  • Provide clear error messages for invalid patches
  • Auto-save patches to browser localStorage
  • Export/import for sharing

ESP32 Runtime Considerations ​

  • Use ArduinoJson for parsing (efficient, well-tested)
  • Pre-allocate maximum buffers to avoid fragmentation
  • Cache parsed parameters (don't parse JSON every audio block!)
  • Provide serial debug output for parsing errors
