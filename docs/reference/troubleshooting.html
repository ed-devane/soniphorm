Troubleshooting ​

Common issues and how to resolve them.

Connection Issues ​

No serial port appears in the browser dialog ​

  • Make sure you're using Chrome or Edge (Web Serial is not supported in Firefox/Safari)
  • Check the USB cable is data-capable (charge-only cables won't work)
  • Close any other application using the serial port (Arduino IDE, PlatformIO, serial monitors)
  • Try pressing the RESET button on the board
  • Try a different USB port on your computer

Connection drops or becomes unresponsive ​

  • Use a shorter USB cable or avoid USB hubs
  • The board may have crashed -- press RESET and reconnect
  • High CPU patches (90%+) can cause serial to become slow

Board doesn't appear after flashing ​

  • Press the RESET button -- ESP32-S3 often needs a manual reset after firmware upload
  • If that doesn't work, enter bootloader mode (hold BOOT + press RESET) and reflash

Audio Issues ​

No sound ​

  • Check that your patch includes a Stereo Out module connected to your audio chain
  • Verify headphones or speakers are plugged into the audio output jack
  • Check the serial console (STATUS command) -- is a patch loaded?
  • Make sure module gain parameters aren't at zero

Clicking or glitching audio ​

  • Check CPU usage with the PERF serial command
  • Reduce module count or switch to lighter alternatives
  • Remove unused modules (each module uses CPU even if not connected)
  • Reverb in "Ultra" quality mode uses significant CPU -- try "Standard"

Hum or noise ​

  • Use a quality USB power supply (avoid cheap chargers)
  • If using Stereo In, check your input source levels

Patch Issues ​

Patch upload fails ​

  • Ensure the serial connection is active (connected indicator in toolbar)
  • Try disconnecting and reconnecting
  • Check the serial console for error messages
  • Very large patches may need multiple upload attempts

Parameters don't respond to pots ​

  • Check you're on the correct bank (turn encoder to switch)
  • Remember soft takeover -- the pot must cross the stored value before it takes effect
  • Verify the pot module is connected to the correct parameter input

SD Card Issues ​

SD card not detected ​

  • Format the card as FAT32 (not exFAT or NTFS)
  • Use a card 32GB or smaller for best compatibility
  • Re-seat the card and press RESET

Sample upload fails ​

  • Check CPU usage -- high CPU (93%+) can interfere with file transfers
  • The transfer uses slower timing at high CPU load -- wait for it to complete
  • Try uploading fewer samples at once

Recovery ​

Board is completely unresponsive ​

  1. Unplug the USB cable
  2. Hold the BOOT button on the board
  3. While holding BOOT, plug in the USB cable
  4. Release BOOT -- the board is now in bootloader mode
  5. Flash the firmware using the patcher's firmware update tool
  6. Press RESET after flashing
